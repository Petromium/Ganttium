### **Context-Aware Right Rail/Sidebar Widgets Requirements**

#### **Objective**
Implement a dynamic, context-sensitive right sidebar ("rail") system that surfaces relevant widgets based on the currently selected page/module from the main (left) sidebar. The design and behaviors must be consistent, highly usable, and align with best practices for enterprise PMIS systems.

***

#### **Core Widgets from Screenshots & Expected Context Awareness**

**1. Portfolio/Project Signals**
- Sprint/work days left, budget burn (%), resource availability
- Always visible on overview/dashboard, and surface on relevant project pages

**2. Upcoming Events/Calendar Summary**
- Meetings, reviews, deadlines/approvals; sync with project calendar context
- Must update based on selected project, workspace, or stakeholder
- Use timezone (Asia/Riyadh) context for event display

**3. Legend**
- Visual identifiers for meetings, reviews, milestones, holidays, etc.
- Should update and filter based on active module (e.g., events, WBS, risks)
- Maintain legend consistency across project modules

**4. Resource/Cost/Risk Snapshots**
- Resource utilization, allocations, cost exposure, risk scores
- Surfaces on cost, resource, and risk management pages only
- Zero-value/empty states must be handled cleanly

**5. Review/Cadence Modules**
- Approvals, steering brief schedules, cadence routines for the workspace
- Calendar widgets must reflect dynamically based on selected phase/active workspace

**6. WBS & Linkage**
- Automatically show WBS linkage and references only when tasks/events tie into WBS nodes

***

#### **Industry-Standard Implementation Points**

- **Modular Sidebar Components:**
  - Each widget must be a modular React component (suggested: Card-based, collapsible, with context props).
  - React + Tailwind CSS recommended; accessibility (ARIA) and responsive design mandatory.

- **Context Management:**
  - Use a centralized context store (React Context API, Redux, or equivalent) to propagate sidebar contextâ€”selected project, tab, or page.
  - Side-rail must update immediately as the left sidebar state changes.

- **Widget Visibility Logic:**
  - Widgets render only if relevant context (page or data) is active.
  - Fallback to empty-state visuals, *never show stale or unrelated widget content*.

- **Data Binding:**
  - All widgets must bind to live data (API, not hardcoded) reflecting real-time state.
  - For signals (budget burn, sprint days): auto-refresh every 15 min (see screenshot standard).

- **Styling and UX:**
  - Consistent margin and padding with floating/elastic behavior.
  - Sidebar must not block main workspace; collapse or overlay modes are required for tablet/mobile.

- **Testing and QA:**
  - Every widget must include unit tests (Jest/React Testing Library).
  - End-to-end interaction test: switching left sidebar routes triggers correct widget updates.
